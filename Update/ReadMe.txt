========================================================================
    控制台应用程序：Update 项目概述
========================================================================

 功能描述：后台更新程序，定时获取最新版本的程序并应用到本地

 实现思路：
	1、访问URL 获取文件清单，清单格式为(FilePath, FileMd5Hash)；
	2、如果存在清单上的文件的MD5与本地文件的MD5不一致；
	3、获取清单上的文件(一个或多个)并保存到本地临时目录，并计算下载的文件MD5是否与其清单中的MD5一致，
		如果不一致，则说明更新服务器上的文件被“恶意篡改”，此时应该向开发人员发送一封Email通知，并且
		停止本次更新，直接跳转到 6。
	4、找到主程序的PID，通过发送信号(SIGINT)给主程序，请求其停止服务
	5、等到主程序停止服务后，替换最新文件
	6、重新启动主程序 CloudMonitor.exe


注意，对于第4步，停止主程序运行的方式解释如下：

	直接在“控制管理器”中结束主程序并不能够完全终结本程序。
	本程序功能庞杂，为降低程序的复杂性，借鉴了 *nix 开发思想，将程序分割为几个独立的部分，
	每一个部分完成各自的功能。
	因此，采用任何形式的终止 CloudMonitor.exe(网络通信)、MonitorService.exe(I/O监听) 进程操作，
	并不能够完全停止本服务。并且有可能会导致用户的windows系统崩溃。



应用程序向导已为您创建了此 Update 应用程序。

本文件概要介绍组成 Update 应用程序的每个文件的内容。


Update.vcxproj
    这是使用应用程序向导生成的 VC++ 项目的主项目文件，其中包含生成该文件的 Visual C++ 的版本信息，以及有关使用应用程序向导选择的平台、配置和项目功能的信息。

Update.vcxproj.filters
    这是使用“应用程序向导”生成的 VC++ 项目筛选器文件。它包含有关项目文件与筛选器之间的关联信息。在 IDE 中，通过这种关联，在特定节点下以分组形式显示具有相似扩展名的文件。例如，“.cpp”文件与“源文件”筛选器关联。

Update.cpp
    这是主应用程序源文件。
